# 01 - 重构计划与执行顺序

## 目标

将 `wallet-flutter` 从演示级 UI 升级为“Rabby 功能域对齐版”，覆盖：

1. 钱包管理（创建/导入/多钱包/切换）
2. 资产与网络管理（代币显示、隐藏，自定义网络）
3. 交易流（发送、接收、历史）
4. DeFi / Swap / NFT / Portfolio 模块
5. dApp 模块（连接管理 + 内置浏览器）
6. 安全与设置（主题、清理、基础安全提示）

## 分阶段

### 阶段 A：架构与数据层
- 统一模型定义（钱包、网络、交易、dApp 书签/历史）
- 强化全局状态 `WalletAppState`
- 本地持久化：Secure Storage + Shared Preferences

### 阶段 B：UI 与主题重构
- 绿色系品牌风格重构（不刺眼）
- 首页 Hero 卡片 + 模块入口重排
- 通用组件标准化（卡片、空状态、页面壳）

### 阶段 C：功能完整度
- 新增钱包管理页、网络管理页、代币管理页
- 交易发送校验增强
- dApp 浏览器实现（WebView）

### 阶段 D：验证与文档
- `flutter analyze`
- `dart test`（纯模型测试）
- iOS 模拟器运行验证
- 输出测试报告与后续接入计划

## 注意

当前实现属于“可运行产品框架 + 关键流程闭环 + 链上能力预留接口”。
真实主网交易、签名、授权扫描等生产能力需继续接入真实 API/SDK。
