# Release Readiness Checklist

## 1. 使用规则

- 任一 P0 条目未通过：禁止发布。
- 发布审批必须附验收证据。

## 2. Beta 门禁

### 2.1 功能完整性（P0）

- [ ] 登录/重连/失效处理可用
- [ ] 市场列表/详情/交易路径完整
- [ ] Portfolio/Activity 可用
- [ ] 结算/索赔路径可用

### 2.2 交易体验（P0）

- [ ] 交易四态可见（pending/confirmed/indexed/failed）
- [ ] 失败可恢复或重试
- [ ] 历史记录可追溯

### 2.3 稳定性（P0）

- [ ] API 可用性 >= 99.5%
- [ ] 7天窗口 P0 故障 = 0
- [ ] 指标/日志/告警可用

### 2.4 安全风控（P0）

- [ ] 鉴权策略生效
- [ ] 限流策略生效
- [ ] 关键行为审计覆盖
- [ ] 异常告警生效

### 2.5 工程质量（P1）

- [ ] 单元/集成/E2E 回归通过
- [ ] CI 门禁通过
- [ ] 回滚预案可执行

## 3. Production 门禁

### 3.1 安全与风险（P0）

- [ ] 安全评审签署通过
- [ ] 风控规则上线
- [ ] 应急SOP演练完成

### 3.2 运维与治理（P0）

- [ ] 值班机制上线
- [ ] 告警分级与升级路径确定
- [ ] 事故模板与复盘模板就绪

### 3.3 发布能力（P0）

- [ ] 灰度发布通过
- [ ] 回滚演练通过
- [ ] 版本兼容性通过

## 4. 发布结论模板

- 版本：
- 阶段：Beta / Production
- P0 结果：通过 / 不通过
- 风险说明：
- 回滚方案：
- 审批人：
