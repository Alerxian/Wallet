# Prediction Market App Product Master Roadmap

## 1. 文档目的

定义从当前 `Demo+` 到可直接发布 `Production` 的完整产品路线、阶段目标、验收标准与里程碑。

## 2. 当前定位

- 当前定位：`Demo+`
- 说明：链路可演示（登录/市场/买卖）但不满足发布级要求（安全、稳定、风控、体验闭环、运维治理不足）。

## 3. 产品目标（北极星）

构建“完整功能、流程闭环、可发布运营、界面美观整洁”的移动端预测市场应用。

## 4. 阶段模型

### Phase 0: Demo+（当前）

- 目标：演示链路可跑通
- 退出条件：
  - 钱包连接可用
  - 市场列表/详情可用
  - 买卖交易可发起

### Phase 1: MVP

- 目标：可小范围真实用户试用
- 必达能力：
  - 会话治理（重连、过期、切换）
  - 交易状态闭环（pending/confirmed/indexed/failed）
  - Portfolio + Activity 闭环
  - 基础风控（鉴权、限流、审计、告警）
  - UI 状态完整（加载/空/错误）
- 退出条件：
  - 核心路径成功率 >= 98%
  - 交易状态可追踪率 = 100%
  - 关键 API 错误率 < 1%

### Phase 2: Beta

- 目标：可灰度发布
- 必达能力：
  - 监控、日志、告警、追踪齐全
  - 自动化回归与 CI 门禁
  - 灰度与回滚演练
- 退出条件：
  - API 可用性 >= 99.5%
  - 7 天窗口 P0 故障 = 0
  - 回滚演练通过

### Phase 3: Production

- 目标：可正式发布与运营
- 必达能力：
  - 完整安全风控策略
  - 应急响应 SOP 与值班机制
  - 发布治理和运营支持
- 退出条件：
  - 发布门禁全绿
  - 安全评审通过
  - 运维交接完成

## 5. 12周里程碑

- W1-W2：规划冻结 + 规格冻结
- W3-W6：MVP 核心功能闭环
- W7-W8：风控与可观测
- W9-W10：Beta 工程化与发布治理
- W11-W12：Production 发布准备

## 6. 完成定义（DoD）

每项需求必须同时满足：

- 功能：用户路径可执行
- 工程：测试通过、日志可定位
- 体验：加载/空/错状态完整
- 文档：需求与验收证据更新
- 发布：可监控、可回滚

## 7. 核心风险

- WalletConnect 兼容性与网络环境差异
- 索引延迟导致交易反馈滞后
- 风控与告警不足导致故障发现延迟
- UI 不一致影响用户信任

## 8. 组织与职责（建议）

- Product：需求优先级、验收标准
- Mobile：用户体验闭环
- Backend：交易与数据正确性
- Contracts：协议安全边界
- QA/DevOps：质量门禁与发布治理
