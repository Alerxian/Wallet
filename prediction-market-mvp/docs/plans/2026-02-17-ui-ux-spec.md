# UI/UX Specification (Cyan Clean Production Style)

## 1. 目标

实现“美观整洁、可信、可发布”的移动端体验，避免 Demo 感，建立一致设计语言。

## 2. 视觉原则

- 主色：Cyan（不偏紫）
- 风格：简洁、金融可信、信息密度适中
- 重点：信息层级清晰、状态表达明确、交互可恢复

## 3. Design Tokens v1

### 3.1 Colors

- `primary`: `#0EA5A8`
- `primary-strong`: `#0B8B8E`
- `bg`: `#F4FBFB`
- `surface`: `#FFFFFF`
- `text-primary`: `#0F172A`
- `text-secondary`: `#475569`
- `success`: `#16A34A`
- `warning`: `#D97706`
- `danger`: `#DC2626`
- `border`: `#D7E6E8`

### 3.2 Typography

- 中文：`Noto Sans SC`
- 英文数字：`Space Grotesk`
- H1 28/34 700
- H2 22/30 700
- Body 15/22 400
- Caption 12/18 500

### 3.3 Spacing & Radius

- Spacing：4/8/12/16/20/24/32
- Radius：8（控件）/12（卡片）/16（弹层）

## 4. 信息架构（MVP）

- Markets（列表）
- Market Detail（详情 + 交易）
- Portfolio（持仓）
- Activity（历史）
- Settings（连接、网络、后端地址、帮助）

## 5. 组件规范

- MarketCard
- TradePanel（Buy/Sell + YES/NO + Amount）
- TxStatusChip（pending/confirmed/indexed/failed）
- BalanceRow / PositionRow
- EmptyState / ErrorState / LoadingState
- ActionButton（default/pressed/disabled/loading）

## 6. 页面状态要求

每个页面必须具备：

- 加载态
- 空态
- 错误态（包含可执行下一步动作）

## 7. 交互规范

- 关键操作二次确认（大额、全部卖出）
- 操作反馈 <= 100ms
- 错误信息可读且可恢复
- 动效节制：只保留必要转场和状态反馈

## 8. 可访问性

- 对比度达到 WCAG AA
- 点击热区 >= 44x44
- 字体缩放兼容
- 不以颜色作为唯一状态信号

## 9. MVP 验收

- 核心页面状态覆盖率 = 100%
- UI 一致性评审 >= 90/100
- 核心流程可用性测试通过率 >= 95%
